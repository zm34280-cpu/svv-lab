# Test Plan

## 1. Introduction
This document describes the testing strategy for the Student Management System.

## 2. Objective
To verify that the system meets all specified requirements.

## 3. Scope
Testing will include:
- User registration
- User login
- Profile management
- Dashboard loading time
- Email notification system

## 4. Test Strategy
- Functional Testing
- Performance Testing
- Security Testing

## 5. Test Cases

### Test Case 1: User Registration
Input: Valid email and password
Expected Result: User account is created successfully.

### Test Case 2: User Login
Input: Registered email and password
Expected Result: User is logged in successfully.

### Test Case 3: Dashboard Load Time
Condition: Normal internet (10 Mbps)
Expected Result: Dashboard loads within 3 seconds.

### Test Case 4: Email Notification
Condition: Successful registration
Expected Result: Email sent within 1 minute.
